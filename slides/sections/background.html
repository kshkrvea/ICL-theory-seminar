- Looking ahead, Tabular foundation models has changed that.

Background

Statistical Model

\(X \in \mathcal{X}\) – features;   \(Y \in \mathcal{Y}\) – classification label

\[ \text{r.v. }\mathcal{D}_n = \big(X^{(i)}, Y^{(i)}\big)_{i=1}^n \quad \text{is s.t. } \big( X^{(i)}, Y^{(i)} \big) \overset{\text{iid}}{\sim} p_0(\cdot, \cdot) \]
-> quickly introduce notation:

-> X is the feature space

-> Y is the label space

-> D_n is the training dataset of n data points coming from p0

-> p0 is the true data generating distribution.

Background

Supervised Learning Framework

At inference we want to estimate \(p_0(Y \mid X)\):

  1. fix \(\mathcal{D}_n = D_n\)
  2. train \(q_\theta(Y \mid X) \approx p_0(Y \mid X)\) on data from \(D_n\)
  3. predict \(\mathbb{P}(Y = \hat{y}_{\text{test}} \mid X = x_{\text{test}}) = q_\theta(\hat{y}_{\text{test}} \mid x_{\text{test}})\)
-> In classical SL setup we assume p0 is fixed and we're gonna to learn it.

-> Train a model on data, make predictions

-> For data from another distribution, retrain the model.

Background

Bayesian Framework

For every \(n\), the statistical model gives a tuple \( (\mathcal{D}_n \cup (Y, X),\, p_0). \)

We want to approximate \(p_0(Y \mid X)\) by

\[ \pi(y \mid x, D_n) \overset{\text{def}}{\equiv} \mathbb{P}\big(Y = y \mid X = x,\, \mathcal{D}_n = D_n\big) \]

With

\[ q_\theta(Y \mid X, \mathcal{D}_n) \approx \pi(Y \mid X, \mathcal{D}_n) \]

And make predictions

\[ \mathbb{P}(Y = \hat{y}_{\text{test}} \mid X = x_{\text{test}}, \mathcal{D}_n = D_n) = q_\theta(\hat{y}_{\text{test}} \mid x_{\text{test}}, D_n) \]
\(\pi\) is called the Posterior Predictive Distribution
-> In bayesian approach we want to ammortize p0 with a function pi, which takes the training dataset as a conditioned input.

-> And by parametrising this model,

-> We can make predictions for new test points without retraining the model.

-> Think what pi should intuitively do implicitly? "Extract" information about relation x-to-y from the training dataset and "apply" it to the new test points.

Background

Meta Level

Prior \(\mathbb{P}_\Phi\)

Define a space \(\Phi\) of all possible DGPs, where each hypothesis \(\varphi \in \Phi\) is s.t. \((x, y) \sim \varphi\).

The distribution \(\mathbb{P}_\Phi\) is called prior and expresses our beliefs about \(\varphi\) before seeing any data.

-> To define this mechanism, define a space of these relationships, i.e. hypothesis space Phi.

-> LHS: "uniform" prior over the hypotheses.

-> RHS: hypothesis "likelihood" after seeing the training dataset D_n.

-> Recall slide 6 for the data example

Background

Posterior Predictive Distribution

\[ \pi(y \mid \htmlClass{ppd-xa}{x},\; D_n) = \int \htmlClass{ppd-int}{\varphi(y \mid x)}\, \htmlClass{ppd-meas}{d\mathbb{P}_{\Phi}} (\varphi \mid \htmlClass{ppd-xb}{x,\;} D_n) \tag{1} \]
\(=\;\) \(\mathbb{P}\big(\varphi = \varphi_1 = \) \(\mid \htmlClass{ppd-xb}{x,\;} D_n\big)\) \(\;\cdot\;\) \(\varphi_1(y \mid x)\)
\(+\;\) \(\mathbb{P}\big(\varphi = \varphi_2 = \) \(\mid \htmlClass{ppd-xb}{x,\;} D_n\big)\) \(\;\cdot\;\) \(\varphi_2(y \mid x)\) \(\;+\;\dots\)
Assumption
\(\mathbb{P}(\varphi \mid x, D_n) = \mathbb{P}(\varphi \mid D_n)\)   i.e. \(x \equiv x_{\text{test}}\) is not informative about hypothesis \(\varphi\).
-> Oce we have this space, let's integrate over it to get a ppd.

-> PPD is a distribution and you can think about it as a weighted sum of hypotheses
with weights given by likelihood of each hypothesis given the training dataset.

-> The assumption is that the test point is not informative about the hypothesis.
i.e. all info is in Dn

Background

Posterior Predictive Distribution

Dropping \(x\) from the conditioning on \(\varphi\):

\[ \pi(y \mid x, D_n) = \int \varphi(y \mid x)\, d\mathbb{P}_{\Phi}(\varphi \mid D_n) = \int p(y \mid x, \varphi) \cdot \underbrace{p(\varphi \mid D_n)}_{\text{not available}} \, d\varphi \]

By Bayes' rule, \(p(\varphi \mid D_n) \propto p(D_n \mid \varphi)\, p(\varphi)\), so

\[ \pi(y \mid x, D_n) \;\propto\; \int \textcolor{olive}{\underbrace{\textcolor{black}{p(y \mid x, \varphi)}}_{\substack{\text{prediction based} \\ \text{on hypothesis } \varphi}}} \cdot \textcolor{teal}{\underbrace{\textcolor{black}{p(D_n \mid \varphi)}}_{\substack{\text{how likely is} \\ D_n \text{ under } \varphi}}} \cdot \textcolor{blue}{\underbrace{\textcolor{black}{p(\varphi)}}_{\substack{\text{prior belief} \\ \text{about } \varphi}}} \, d\varphi \]
-> The problem of this view is that likelihood is not available.

-> Apply bayes rule and read from right to left.

-> But this object is still theoretical

Background

Prior-Data Fitted Networks (PFNs)

Prior-Data Fitted Network is a numerical approximation of the entire family of PPDs \(\{\pi(\cdot \mid \cdot,\, \mathcal{D}_n)\}_{n}\).

Theorem 2.2.
Let \(\mathcal{Q}\) be the set of all conditional probability functions. Then the PPD satisfies \[ \pi \;=\; \arg\max_{q \in \mathcal{Q}}\; \mathbb{E}_{{\color{blue}\varphi \sim \mathbb{P}_{\Phi}},\, {\color{teal}\mathcal{D}_n \cup (Y, X) \sim (\mathcal{D}_{n+1} \mid \varphi)}} \big[\log {\color{olive}q(Y \mid X, \mathcal{D}_n)}\big]. \]

\(\Downarrow\) restrict to a parametric model \(q_\theta\)

\[ \theta^{*} = \arg\max_{\theta}\; \mathbb{E}_{N \sim \mathbb{P}_N,\, {\color{blue}\varphi \sim \mathbb{P}_{\Phi}},\, {\color{teal}\mathcal{D}_N \cup (Y, X) \sim (\mathcal{D}_{N+1} \mid \varphi)}} \big[\log {\color{olive}q_{\theta}(Y \mid X, \mathcal{D}_N)}\big] \]

\(\Downarrow\) approximate in practice by Monte-Carlo

\[ \widehat{\theta} = \arg\max_{\theta}\; \sum_{j=1}^{m} \log q_{\theta}\big(Y_j \mid X_j, D^{(j)}\big) \]
-> To get it in practice, we may use result of the theorem from T.Nagler's paper:

-> It gives us a form of a loss function to optimize.

-> So in practice we parameterize the model and optimize that loss.

-> Basically PFN == special case of a neural process:
- invariant to permutations of both context and test dataset samples
- models PPD.

-> Optimize not ELBO but log-likelihood, thanks to acess to the data prior via proposed DGP.
